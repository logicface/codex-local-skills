# Experiment Analysis

## Core files

- Postrun watcher and publisher: `tools/postrun_watch_and_publish.py`
- Plotting and summaries: `tools/plot_regression_run_summary.py`, `tools/plot_odi_regression.py`
- Paper figures: `tools/paper_plot_epoch_trajectory.py`, `tools/paper_plot_fnirs_quicklook.py`, `tools/paper_plot_fnirs_success.py`, `tools/paper_plot_medical_figures.py`
- Audits: `tools/run_gain_predictability_audit.py`, `tools/run_quality_audit.py`
- Experiment notes: `docs/experiments/`
- Auto-generated analyses and dashboards: `docs/experiments/auto/`

## What to inspect

- Run directory and host log
- Summary JSON or markdown generated by the run
- Metrics history over epochs
- Checkpoint-specific metric blocks
- Prediction histogram skew or collapse warnings

`tools/postrun_watch_and_publish.py` already contains logic for:

- waiting for training completion
- locating the latest run directory
- loading metrics rows
- generating markdown analysis
- detecting dominant prediction collapse patterns

Use that before writing a parallel parser.

## Validation checklist

- Confirm the run you are analyzing is the intended one.
- Distinguish final epoch metrics from best-checkpoint metrics.
- If comparing runs, normalize for dataset and split.
- If making a paper figure, record the exact script and input artifacts used.
- If a run looks collapsed, cite the histogram or checkpoint evidence directly.
