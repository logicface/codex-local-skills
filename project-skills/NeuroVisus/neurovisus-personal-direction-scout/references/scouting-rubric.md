# Scouting Rubric

## 1. Infer the lab spine

Before inventing directions, summarize the current workstream in compact terms:

- target task
- main modalities
- current baseline ladder
- most active improvement theme
- current bottleneck

Use repository evidence first, then the user's stated group direction.

## 2. Inventory the user's local leverage

Estimate or ask for:

- strongest technical edge: modeling, data cleaning, analysis, clinical interpretation, systems work
- realistic time horizon
- publication target
- tolerance for risky directions
- whether the user needs a fast publishable line or a longer-term thesis spine

If the user did not provide this, state the assumption explicitly.

## 3. Score each candidate direction

Score candidate lines against:

- `repo fit`: can the first experiment happen in the current codebase
- `scientific distinctiveness`: is it more than a minor sweep
- `baseline clarity`: can it beat or explain the current baseline ladder
- `dependency risk`: does it require unavailable data, labels, or infra
- `paper path`: workshop, strong conference, journal increment, or thesis chapter potential
- `personal ownership`: can the user reasonably claim it as their own line

## 4. Candidate template

For each candidate direction, write:

1. `Direction`
2. `Why it fits the group`
3. `Why it can become the user's own line`
4. `Minimal first experiment in NeuroVisus`
5. `Likely failure mode`
6. `What success would look like`
7. `Best publication path`

## 5. Portfolio structure

Prefer a balanced portfolio:

- `conservative line`: likely executable and publishable soon
- `balanced line`: good tradeoff of novelty and executability
- `high-upside line`: riskier but potentially more differentiating

Recommend one as primary and one as backup.

## 6. Roadmap shape

### 3-month roadmap

Include:

- reading or evidence consolidation
- baseline reproduction or cleanup
- first discriminative experiment
- first go/no-go checkpoint

### 6-month roadmap

Include:

- stronger version of the primary line
- one backup branch if the primary line fails
- target artifact: benchmark, ablation package, paper draft, or thesis chapter seed
